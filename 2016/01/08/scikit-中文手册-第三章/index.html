scikit 中文手册 第三章

使用距离来建立模型

简介

在这一节，我们开始考虑集群。集群一般使用在非监督学习。这些技术假设我们不知道输出。集群是个很有用的设定。这可能是为什么集群是很有效率的。这可以处理一些问题。

使用KMean来聚集数据

聚类是一种十分有用的技术，考虑把潜在的客户分离，分类器帮助我们完成这一过程。

准备

首先我们看看简单的聚类，然后看看KMean是如何工作的：

from sklearn.datasets import make_blobs
blobs, classes = make_blobs(500, centers=3)

怎样做

我们将要举一个简单的例子，从错误的数据。接下来我们来谈谈工作原理和如何优化它。

f, ax = plt.subplots(figsize=(7.5, 7.5))
ax.scatter(blobs[:, 0], blobs[:, 1], color=rgb[classes])
rgb = np.array(['r', 'g', 'b'])
ax.set_title("Blobs")

我们使用KMean来找到这些聚类的中心。

from sklearn.cluster import KMeans
kmean = KMeans(n_clusters=3)
kmean.fit(blobs)
KMeans(copy_x=True, init='k-means++', max_iter=300, n_clusters=3,
n_init=10, n_jobs=1, precompute_distances=True,
random_state=None, tol=0.0001, verbose=0)
kmean.cluster_centers_
array([[ 0.47819567, 1.80819197],
[ 0.08627847, 8.24102715],
[ 5.2026125 , 7.86881767]])
f, ax = plt.subplots(figsize=(7.5, 7.5))
ax.scatter(blobs[:, 0], blobs[:, 1], color=rgb[classes])
ax.scatter(kmean.cluster_centers_[:, 0],
kmean.cluster_centers_[:, 1], marker='*', s=250,
color='black', label='Centers')
ax.set_title("Blobs")
ax.legend(loc='best')

我们使用labels_属性来看看每一个点的标签。

 kmean.labels_[:5]
array([1, 1, 2, 2, 1], dtype=int32)

transform函数十分有用，会输出每一个到中心的距离。

kmean.transform(blobs)[:5]

怎样工作

KMean是一个十分简单的算法，它最小化平方距离的平均值，我们将最小化平方和。

优化重心的数量

要决定我们应该使用多少个重心是一个很难的问题。我们应该明白数据是不平衡的所以会影响评估方法。

准备

评估模型的表现对于非监督模型是很有挑战的，sklearn有很多方法评估聚类的，但是用的不多。

怎样去做

我们还是要模拟数据聚类这个过程。

from sklearn.datasets import make_blobs
import numpy as np
blobs, classes = make_blobs(500, centers=3)
from sklearn.cluster import KMeans
kmean = KMeans(n_clusters=3)
kmean.fit(blobs)
KMeans(copy_x=True, init='k-means++', max_iter=300, n_clusters=3,
n_init=10, n_jobs=1, precompute_distances=True,
random_state=None, tol=0.0001, verbose=0)

首先，我们来看看轮廓距离。它是不同的比率归类和未归类的比较。
from sklearn import metrics
silhouette_samples = metrics.silhouettesamples(blobs,
kmean.labels)
np.column_stack((classes[:5], silhouette_samples[:5]))
f, ax = plt.subplots(figsize=(10, 5))
ax.set_title(“Hist of Silhouette Samples”)
ax.hist(silhouette_samples)
这里的数据会逐渐升高直到1.

怎样工作

我们来看看轮廓距离的平均值，来描述它：
silhouette_samples.mean()
这个十分普遍，实际上这个度量模块表现的函数：

metrics.silhouette_score(blobs, kmean.labels_)
    0.57130462953339578

我们计算多个分类，来看看平均数值：

first new ground truth
blobs, classes = make_blobs(500, centers=10)
sillhouette_avgs = []
this could take a while
for k in range(2, 60):
kmean = KMeans(n_clusters=k).fit(blobs)
sillhouette_avgs.append(metrics.silhouette_score(blobs,kmean.labels_))
f, ax = plt.subplots(figsize=(7, 5))
ax.plot(sillhouette_avgs)

我们可以看出根据这个图应该有6或者7个中心，但是其实我们知道只有三个中心，我们只能盼望检测的数量比较接近。

接触聚类的正确性

我们来谈论有关聚类的正确性。

准备

我们假设一种情况我们使用外部代理来处理真实数据。我们使用简单的数据集，评估这些方法的正确性。我们创造简单的数据集，来讨论这个。
from sklearn import datasets
from sklearn import cluster
blobs, ground_truth = datasets.make_blobs(1000, centers=3,cluster_std=1.75)

怎样去做

在我们处理这些之前，让我们看看这个数据集。
f, ax = plt.subplots(figsize=(7, 5))
colors = [‘r’, ‘g’, ‘b’]
for i in range(3):
p = blobs[ground_truth == i]
ax.scatter(p[:,0], p[:,1], c=colors[i],label=”Cluster {}”.format(i))
ax.set_title(“Cluster With Ground Truth”)
ax.legend()
f.savefig(“9485OS_03-16”)
